## 0. 一句话定义

**用户完成打卡训练后,生成一张视觉精美、转发得体的「成就卡片」图片,可保存到相册或通过系统分享面板发出去;靠卡片自身的传播力带来品牌曝光与新用户,无需任何后端。**

对标:多邻国 streak 截图、Keep 完成卡、背单词 App 的「打卡图」。

---

## 1. 背景与产品判断

### 1.1 为什么是「成就卡片」而不是「邀请返利」

Tilemo 当前是 **0 后端、纯离线、私密** 的产品(数据全在本地,键名 `tgm:*`,不登录不上传)。
传统「裂变」(邀请码 / 归因 / 返利阶梯)需要账号系统 + 服务端 + 数据库,会**直接摧毁这个核心定位**——一旦要登录上传,「私密」这个故事就没了。

因此本功能选择 **靠内容驱动的病毒式传播**:把用户的「坚持」变成一张值得发朋友圈的图。每张被分享出去的卡片,就是一次免费的、自带信任背书的品牌广告。

### 1.2 最关键的产品判断:**品类的「转发得体性」**

提肛 / 凯格尔 / 盆底肌训练是**隐私敏感的健康品类**。用户**不会**愿意在朋友圈直白地写「我练提肛 30 天」——这会让人尴尬。

所以卡片的文案与视觉必须做到**「圈外人看也觉得优雅、圈内人才懂」**:

- ❌ 不出现「提肛 / 凯格尔 / 盆底肌」这类直白词
- ✅ 用抽象的「坚持 / 兑现承诺 / 一格一格亮起来」来表达成就感
- ✅ 用品牌双关 slogan「今天,提了。」——圈内人懂是「提肛」,圈外人只看到一个很酷的打卡 App

**「今天提了么」这个 slogan 本身就是最好的分享钩子**:它让分享者保全体面,又勾起旁观者的好奇。这是本功能能成立的前提,也是文案策略的总纲。

### 1.3 复用已有资产,不新增视觉语言

产品的 `design-tokens`(blob 圆角、珊瑚红 `#E85A4F`、热力图色阶 `heat0–3`)和「一格一格亮起来」的热力图隐喻,**本身就是天然的卡片视觉语言**。卡片不需要发明新设计,而是把产品已有的视觉签名装进一张图里。

## 2. 目标与非目标

### 目标
- **G1**:用户完成训练后,能在 2 次点击内生成并保存 / 分享一张成就卡片。
- **G2**:卡片在脱离 App 语境(朋友圈 / 微信对话)时依然「转发得体」。
- **G3**:卡片成为产品最主要的自然增长来源(无付费推广前提下)。
- **G4**:零后端、零账号改动上线。

### 非目标(本期不做)
- ❌ 邀请码、归因、返利阶梯(需后端,破坏定位)
- ❌ 社交关系 / 好友排行榜(同上)
- ❌ 服务端分享链接统计(改用本地埋点 + 落地页访问量,见 §8)
- ❌ 付费推广、投放

## 3. 用户与场景

| 用户 | 场景 | 分享欲强度 |
|---|---|---|
| 新用户 | 第 1 次完成训练 | 中 —— 「我开始了」 |
| 坚持用户 | 连续 7 / 30 / 100 天 | **高** —— 里程碑是分享欲峰值 |
| 日常用户 | 每天完成一组 | 低 —— 不能每次都弹,会打扰 |
| 老用户 | 月底 / 年底 | **高** —— 「我的坚持地图」天然适合回顾分享 |

**结论:触发点要分层。日常克制提供,里程碑强推,回顾常驻入口。**(详见 §5)

---

## 4. 核心流程

```
完成一组训练(Metronome done 阶段)
        │
        ▼
[完成态] 现有「完成 + 好的」
        │  + 新增「分享这次」次级按钮(克制,不自动弹)
        ▼
{ 分享卡片浮层 }
   ├─ 预览卡片(实时,带热力图 + 文案)
   ├─ [换一句] 换配文(QUOTES 轮换)
   ├─ [保存图片] → 写入相册(mobile)/ 下载(web)
   └─ [分享]     → 系统分享面板(微信/朋友圈/...)
        │
        ▼
本地埋点:card_generated / card_saved / card_shared +1
```

**里程碑触发**:在 `recordSession` 之后,若 `streak.current` 命中 {1, 7, 30, 100, 365} 或累计次数达 {100, 500, 1000},**自动**弹出一张更强仪式感的里程碑卡片(带「Day 30」大字、徽章),而非等用户主动点。

## 6. 卡片设计规格

### 6.1 三种卡片类型

| 类型 | 主视觉 | 主文案 | 适用触发 |
|---|---|---|---|
| **A. 日常完成卡** | 当日单格高亮 + 完成组数进度条 | QUOTES 轮换一句 | 日常完成 |
| **B. 里程碑卡** | 「Day 30」超大字 + 徽章 + 连续天数热力条 | 专属里程碑文案 | 连续天数 / 累计阈值 |
| **C. 回顾卡** | 近 30 天热力图网格(产品核心隐喻) | 「坚持的形状,是一格一格亮起来的」 | 回顾入口 |

### 6.2 视觉规格(来源:`packages/design-tokens`)

- **画布**:1080 × 1350(4:5,社交平台最佳展示比例),也可输出 1080 × 1080 方图。
- **背景**:`paper #FAF3EC`(浅)/ `#1F1814`(深,跟随系统主题)。
- **主色**:`accent #E85A4F`(珊瑚红),用于高亮格 / 徽章 / 品牌。
- **热力图色阶**:`heat1 #F0DCC9` → `heat2 #E8B59A` → `heat3 #E85A4F`,浅 → 深。
- **圆角**:卡片四角用不对称 blob `radius.blob1/2/3`(品牌签名,不要用规整圆角)。
- **字体**:`-apple-system`,大数字用 `font.num`。
- **品牌落款**:底部小字「**今天,提了么**」(中文优先) + 可选英文 Tilemo。**不放 Logo**(项目目前无 Logo 资源,文字 slogan 更高级)。

### 6.3 文案策略(本功能成败关键)

**铁律:卡片正文绝不出现「提肛 / 凯格尔 / 盆底肌」。**

可复用 `QUOTES`(`packages/core/src/quotes.ts`)中天然得体的句子:
- 「坚持的形状,是一格一格亮起来的。」
- 「看不见的肌肉,看得见的坚持。」
- 「今天比昨天多提一组,就赢了。」(「提」双关,圈内懂)
- 「又一天,又一组,又稳了一点。」
- 「你比你以为的更自律。」

里程碑专属文案示例:
- Day 7:「第一周,兑现了。」
- Day 30:「三十天,身体记得。」
- Day 100:「一百天,已经是习惯了。」
- 累计 1000 次:「一千次收紧,慢慢来,肌肉记得住每一次。」

数据元素(可直接拼上卡片):
- `streak.current`(连续天数)
- 今日完成组数 / 目标组数(`today.sessions.length` / `settings.dailyGoalGroups`)
- 热力等级 `levelOfDay`
- 训练计划名(`plan.name`)—— 但**建议弱化或省略**,避免品类暴露

### 6.4 落地页二维码(V2,可选增强)

卡片角落放一个小二维码,指向公开落地页 `https://<web 域名>`(Vercel 已部署)。**这不是裂变归因**(无后端无法归因),只是降低旁观者「这什么 App」的获取门槛。落地页访问量由 Vercel Analytics 衡量(被动、不侵犯用户隐私)。

## 7. 分享与保存能力

### Mobile(`apps/mobile`)
- **生成图片**:新增依赖 `react-native-view-shot`(把卡片 RN View 截成 PNG)。备选:`@shopify/react-native-skia`(更高性能但体积大,首期不必)。
- **保存相册**:`expo-media-library` 写入。
- **系统分享**:`expo-sharing`(已在 `package.json` 声明但**目前闲置**,真正启用它分享文件) + RN `Share`(已在用)。
- 参考:`apps/mobile/src/screens/SettingsScreen.tsx:33-54` 已有 `Share.share` 文本分享模式,可借鉴流程。

### Web(`apps/web`)
- **生成图片**:浏览器原生 **Canvas API** 绘制 → `canvas.toBlob()` → 下载。**零新依赖**。
- 卡片视觉规格与 mobile **完全一致**(同一份 design-tokens),只是渲染载体不同(JSX vs Canvas)。

> 两端要各写一套卡片渲染(mobile JSX,web Canvas),但设计规格单一来源是 `design-tokens`。

## 8. 增长闭环与指标(无后端如何衡量)

没有服务端,「裂变」闭环这样闭合:

```
用户生成并分享卡片 ──┐
                    ├─→ 卡片在社交网络曝光 ──→ 旁观者扫落地页二维码
本地埋点(生成/保存/分享次数)        │
                    │              └─→ 落地页访问量(Vercel Analytics)
                    └──────────────────→ 旁观者下载 App(自然增长)
```

**本地埋点**(新增键 `tgm:share-stats`,纯本地,不上传):
- `card_generated`、`card_saved`、`card_shared` 的累计计数
- 按卡片类型(A/B/C)拆分
- 用途:在 Settings 里给用户看「你已分享 N 次,点亮了 N 格」(正向反馈,鼓励继续分享);**也用于我们后续判断哪种卡片 / 哪个触发点转化最好,迭代设计**。

**被动曝光指标**:Vercel Analytics 看落地页 UV / 来源(无需埋点,不碰用户隐私)。

> MVP 不做埋点也行;埋点是 V1.1。核心闭环(分享 → 曝光 → 落地页)不需要任何埋点即可成立。

## 10. 风险与注意事项

| 风险 | 说明 | 对策 |
|---|---|---|
| **品类尴尬导致没人分享** | 隐私品类,直白文案会让用户拒绝分享 | 文案铁律(§6.3)+ 抽象化视觉,默认卡不暴露品类 |
| **每次打卡都弹分享 → 打扰** | 日常完成分享欲低,强弹会反感 | 日常「克制提供」,只有里程碑自动弹 |
| **图片生成依赖体积 / EAS 打包** | `react-native-view-shot` 增包体 | 体积可接受;若敏感改用 Skia,首期不必 |
| **相册 / 分享权限** | iOS 需相册权限,Android 需存储权限 | 首次保存时申请,文案解释用途 |
| **web 跨域 toBlob** | Canvas 跨域图片会污染画布 | 卡片只用纯色 + 文字 + 矢量格子,不引外部图 |
| **过度承诺增长** | 「裂变」效果取决于用户自发意愿 | 不设 KPI 硬指标,先验证「卡片是否被分享」 |
