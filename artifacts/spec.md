# 今天提了么 — 提肛助手

## 概述

**今天提了么**是一款轻量、私密、可完全离线运行的每日提肛（凯格尔 / 盆底肌训练）打卡与引导训练工具。它的核心不是一个复杂的健身 App，而是一个"每天问你一句、陪你提一组、替你记一笔"的安静小工具——像床头柜上的一颗小石子，存在感低，但天天在。

核心价值三句话：
1. **降低启动门槛**——打开就是今天有没有提、还差几组，不用设置、不用登录、不用联网。
2. **把训练做成节拍器**——提肛的难点不在"会不会"，而在"提多久、放多久、做几组"。App 用可视化的呼吸/收缩节拍引导，让人不用数数也能跟练。
3. **让坚持被看见**——用日历热力和连续天数把"看不见的肌肉训练"变成"看得见的日常痕迹"，但不制造焦虑、不发推送、不催促。

目标用户是任何想养成这个习惯的成年人（久坐者、产后恢复人群、关注盆底健康的人）。考虑到主题的私密性，整个产品的基调必须是**不羞耻、不医疗化、轻松但不轻浮**。

## 视觉设计语言

### 设计概念（一句话）
**"晨光里的呼吸节拍器"**——介于冥想 App 的安静和习惯打卡 App 的清爽之间，用圆润、温润、带一点点弹性的视觉语言，把一组"提起来—松开"的动作变成一件每天值得做的事。

### 情绪与氛围关键词
温润 · 安心 · 克制 · 有弹性 · 私密感

### 色彩方向

避免医疗器械的冷蓝、也避开健身 App 的高饱和荧光。走一条"清晨自然光"的暖色系。

- **主色 / 品牌色**：温润的陶土橘（terracotta / clay）——介于赭石和暖珊瑚之间，不刺眼，有温度。传达"身体、血肉、暖意"，恰好契合盆底肌主题，又不色情。
- **辅助色**：晨雾米白（背景主色）、墨绿（少量点缀，象征"持续/生长"，用于连续打卡成就）、深夜炭（深色模式背景，偏暖的黑，不要纯黑）。
- **中性色**：暖灰梯度（不用纯灰，全部带一丝暖调）。
- **语义色**：成功用墨绿而非俗气的亮绿；今日未完成用陶土橘的浅色变体而非红色警告（不要制造"没做就是错"的压力）。

**明暗基调**：默认浅色（晨光感），提供暖色调深色模式（夜间跟练不刺眼）。两个模式都要做。

### 字体气质

- **中文**：现代无衬线，字重偏中、字形偏圆（如思源黑体的圆润变体方向），避免宋体的严肃感，也避免太卡通的圆体。
- **数字（连续天数、计数、倒计时）**：使用稍带几何感的展示性数字，是页面的视觉锚点——大号、自信、但不张扬。可以用一个偏轻量的等宽或几何无衬线专门渲染数字。
- **西文**：与中文匹配的人文无衬线。

### 排版与布局性格

- **移动优先、单列、纵向叙事**。每一屏只解决一件事，不堆功能。
- **大留白 + 大数字**。文字信息克制，让"今天的数字"和"节拍圆环"成为主角。
- **非对称编辑式排版**：标题左对齐、数字可以出血到边缘，避免"居中三件套"的俗套 hero。
- 卡片用**大圆角**（见 token），但不要全部卡片化——用色块分区代替卡片堆砌。

### 圆角与间距

圆角偏大且统一，传递"柔软、不伤人"的感觉。

### 动效原则

- **一切动效服从"呼吸"这个母题**：节拍器的收缩-放松是核心动效，其余界面动效都和它同频（缓入缓出、不弹跳过猛）。
- 打卡完成的反馈是**一次温柔的脉冲/光晕**，不是放烟花、不是撒纸屑。克制。
- 页面切换用**淡入 + 微下移**，不用滑动堆叠的复杂转场。
- 数字增长（连续天数 +1）用 ease-out 的计数动画，让人"看见"进步。
- 尊重 `prefers-reduced-motion`：节拍器仍可用（改为静态文字提示"收紧 3 秒 / 放松 3 秒"），装饰动效全部关闭。

## 功能与页面

按优先级 P0（必须有）→ P2（锦上添花）排序。每个功能用用户故事描述，不写验收细节。

### P0 — 核心闭环（没有这些就不能用）

**F1. 今日打卡"今天提了么"**
- 作为用户，我想打开 App 第一眼就看到"今天提了么"这个问题以及我今天的完成状态，以便我不需要点进去就知道今天还差多少。
- 作为用户，我想一键完成今日打卡（或一键开始一组训练即视为打卡），以便最低成本地完成今天的任务。
- 作为用户，我想看到今日目标（如"3 组"）和已完成组数，以便知道进度。

**F2. 引导式训练节拍器**
- 作为用户，我想点开"开始训练"后进入一个全屏的节拍引导界面，它告诉我"收紧 N 秒 / 放松 N 秒"，以便我不用自己数数也能跟练。
- 作为用户，我想在节拍界面看到收缩-放松的视觉化（一个呼吸般缩放的圆环 + 文字提示 + 可选的轻微触感/提示音），以便我在任何环境（静音场合 / 不便盯屏）都能跟练。
- 作为用户，我想能选择预设方案（如入门：3 秒收 / 3 秒放 / 10 次；进阶：5 秒收 / 5 秒放 / 15 次），以便按自己的水平训练。
- 作为用户，我想完成一组训练后自动回到今日页并更新进度，以便无缝衔接。
- 作为用户，我想在训练中可以暂停 / 提前结束，并且提前结束也算部分完成，以便我被打断时不至于"白做"。

**F3. 本地记录与连续打卡**
- 作为用户，我想看到我的连续打卡天数（streak）和总打卡天数，以便我有坚持的动力。
- 作为用户，我想看到一个日历视图（月度），上面用颜色深浅标出每天的完成程度（热力图），以便我一眼看到自己的习惯轨迹。
- 作为用户，我想所有数据只存在我的设备上、不上传任何地方，以便保护这个私密主题的隐私。

**F4. 设置与每日目标**
- 作为用户，我想在设置里调整我的每日目标组数、默认训练方案、是否开启提示音/触感、浅色/深色/跟随系统，以便把工具调成适合自己的样子。
- 作为用户，我想能手动补卡/编辑历史某天的记录（用于"今天忘了点但实际做了"的情况），以便记录真实而非教条。

### P1 — 让它更值得每天打开

**F5. 训练方案库 / 自定义方案**
- 作为用户，我想从一个方案列表里挑选（入门 / 巩固 / 进阶 / 快速一组），也可以创建自定义方案（自定义收紧秒数、放松秒数、次数、组数），以便长期保持新鲜感和进步。
- 作为用户，自定义方案存在本地并可命名，以便我记住"这是我常用的那套"。

**F6. 统计与洞察**
- 作为用户，我想看到一个简洁的统计页（本周/本月完成率、最常训练的时段、最长连续天数），以便我了解自己的规律。
- 作为用户，统计页不要过度图表化，保持和全局一致的克制气质。

**F7. 温柔的成就系统**
- 作为用户，我想在达到某些里程碑（连续 3/7/30 天、累计 100/500 组等）时得到一个**低调、不打断**的确认（例如日历上多一枚小印记、成就页里悄悄点亮一项），而不是弹窗轰炸，以便获得正向反馈而不被骚扰。

### P2 — 锦上添花（生成者有余力再做，否则留白）

**F8. 每日一句轻鼓励**（本地内置文案库，随机一句，可关）
**F9. 数据导出/导入**（导出为 JSON 文件本地下载，便于换设备时迁移——纯本地、不联网）
**F10. PWA 安装提示**（让网页端能"添加到主屏幕"变成类 app 体验，这是"移动 app 端"的实现路径之一，见下）

## 两端形态与复用策略

**关键决策：一套静态页面，两个端复用。**

- **网页端**：直接在浏览器打开的单页应用（SPA），所有资源内联或同目录本地文件，无后端、无网络请求。
- **移动 app 端**：理解为"同一套静态页面被装进一个移动 WebView / PWA shell"。具体实现路径由生成者选择，但推荐 **PWA（可安装到主屏幕、支持离线缓存 via service worker 或 manifest）** 作为移动 app 端的首选形态——这样网页端和移动端天然就是同一份代码，零额外维护。
- **响应式策略**：移动优先设计，桌面端用**居中的窄容器（max-width 约 480–560px）+ 两侧留白**呈现，而不是把同一个内容拉伸成宽屏的多列布局。这个产品本质是"手机形态"，桌面只是放大显示，不要为桌面硬造第二套布局。
- 触控优先：所有可点击区域 ≥ 44px，节拍器支持手势（如长按维持收紧）。

## 离线与本地存储约束

### 硬约束（红线）
- **不联网**。整个应用不发起任何 fetch/XHR/WebSocket。图片、字体、图标全部内联（SVG inline / base64 / 系统字体），不引用任何 CDN。
- 数据全部存在浏览器本地：`localStorage`（小而结构化）+ 可选 `IndexedDB`（大记录）。建议优先 localStorage，结构清晰即可。
- 页面可被保存为单一 HTML 文件离线打开（双击即可运行）是加分项。

## 信息架构与页面划分

底部标签式导航（移动）/ 左侧或顶部导航（桌面容器内同样用底部标签以保持一致），四个入口：

1. **今日（Home / 今天提了么）**
   - 顶部：日期 + 每日一句（可关）
   - 主视觉：今日大问题"今天，提了么？" + 进度（已完成 X / 目标 Y 组）
   - 主按钮：开始一组训练 / 已完成今日（脉冲庆祝）
   - 连续天数小徽章
   - 最近 7 天迷你热力条

2. **训练（Train）**
   - 方案选择列表（卡片式，每个方案显示"收紧 Xs / 放松 Ys / N 次 / M 组"）
   - 进入后是**全屏节拍器视图**（独立沉浸态，无导航打扰）：
     - 中央：呼吸般缩放的圆环，环上标"收紧 / 放松"，圆环内大号倒计时数字与当前阶段文字
     - 顶部：第几组 / 共几组、当前第几次 / 共几次
     - 底部：暂停 / 结束
     - 完成后：温柔的完成确认 + 自动回今日页

3. **记录（History）**
   - 顶部：连续天数 + 总天数 + 总组数（三个大数字）
   - 月度日历热力图（可切换月份）
   - 点击某天展开当天详情（做了哪些组、用了什么方案、可编辑）
   - 下方：本周/本月完成率小统计

4. **设置（Settings）**
   - 每日目标组数（步进器）
   - 默认训练方案
   - 提示音 / 触感开关
   - 主题（浅 / 深 / 跟随系统）
   - 每日一句开关
   - 训练方案管理（新增/编辑/删除自定义方案）
   - 数据：导出 JSON / 导入 JSON / 清空全部数据（需二次确认）
   - 关于：一句产品理念 + "全部数据仅存于本设备" 的隐私声明

## 内容清单

### 真实感文案（不要 lorem ipsum）

- **Slogan / 主问句**："今天，提了么？"
- **副标语候选**：
  - "每天一组，悄悄变好。"
  - "看不见的肌肉，看得见的坚持。"
  - "三分钟，给自己的盆底一个交代。"
- **每日一句轻鼓励（本地内置文案库，≥ 20 条，随机展示，可关）**——风格要克制温暖，不鸡汤不油腻，例如：
  - "今天比昨天多提一组，就赢了。"
  - "习惯不是一次立志，是今天又做了一次。"
  - "提起来的时候，别忘了呼吸。"
  - "没人看见，但你自己知道。"
  - "（更多请生成者补足，保持同一气质）"
- **节拍器阶段提示文字**：
  - 收紧阶段："收紧" / "hold — 收紧"
  - 放松阶段："放松" / "放松，正常呼吸"
  - 准备倒计时："准备好了吗？3、2、1"
  - 完成："今天，提了。👍"（emoji 可选/可关，符合"俏皮但克制"）
- **训练方案名称与描述**：见数据模型草稿，每个方案配一句描述（如"入门：从最短的 3 秒开始，建立感觉"）。
- **成就文案**（P1，低调命名）：
  - 连续 3 天："小苗" / 连续 7 天："扎根" / 连续 30 天："深根"
  - 累计 100 组："百组达成" / 500 组："五百次坚持"
  - 全勤一周："这一周没缺席"

### 图片 / 插画方向

- **不使用真人照片**（私密主题，避免任何尴尬或不当联想）。
- **不使用解剖图**（避免医疗化）。
- 主视觉用**抽象图形**：呼吸的圆环、热力色块、有机的弧线、像晨雾或肌肉纤维抽象化的渐变色块。
- 图标用**线条圆润的线性图标**（inline SVG），统一 2px 描边、圆头圆角。
- 如需插画点缀，用**单色 + 品牌色的极简几何有机形**，风格向 Headspace / Calm 的克制插画靠拢，但更几何、更克制。

## 成功标准（达标定义，供 evaluator 参考）

1. **离线可运行**：断网状态下打开、完成打卡、训练一组、查看历史，全部正常，无任何 console 报网络错误。刷新后数据仍在。
2. **核心闭环完整**：今日页看到状态 → 开始训练 → 跟着节拍完成 → 回到今日页看到进度 +1 → 历史页能看到今天的热力点亮。这条链路是产品的脊柱，必须无断点。
3. **视觉一眼可辨**：打开后能感受到"温润、安静、有呼吸感"的整体气质，主色陶土橘 + 暖米背景辨识度高，明显区别于通用模板。**规避俗套清单中的任何一项出现即视为不达标**。
4. **节拍器可用且有沉浸感**：训练时视觉收缩-放松与文字/计时同步，能让人不用数数就跟着做；暂停/提前结束/完成反馈都到位。
5. **数据私密且持久**：所有数据本地存储，刷新/关闭浏览器再打开数据不丢；设置页明确声明数据仅存本设备。
6. **响应式**：手机尺寸和桌面尺寸下都布局合理，桌面端不出现难看的拉伸空白或强行多列。
7. **无障碍底线**：尊重 `prefers-reduced-motion`、`prefers-color-scheme`；可点击区域足够大；文字对比度达标（WCAG AA）。
8. **不堆砌**：P2 功能不实现不算扣分，但 P0/P1 必须扎实。宁可少而精，不要多而散。

## 不做什么（明确排除）

- **不做账号系统 / 登录 / 注册**——纯本地。
- **不做任何联网功能**：无云同步、无社区、无排行榜、无推送通知（推送需要服务端，且与"不打扰"理念冲突）。
- **不做付费 / 订阅 / 广告**。
- **不做医疗诊断 / 健康风险评估 / 建议**——它是习惯工具，不是医疗设备，避免合规风险。
- **不做复杂的多习惯追踪**——只聚焦提肛这一件事，不做成通用习惯 App。
- **不依赖任何 CDN、任何远程字体、任何远程图片、任何第三方 SDK**。连 Google Fonts 都不行，用系统字体或内联字体。
- **不做夸张的庆祝动效、不做"未完成就催你"的焦虑设计**。
- **不做需要数月的复杂课程体系**——保持轻量，方案是参数组合，不是教学内容。

## 给生成者的几句话

这个产品的灵魂在"克制"二字：克制功能数量、克制视觉噪音、克制情绪施压。它不是要成为功能最全的凯格尔 App，而是要成为"每天愿意打开、打开就想做一组、做完就放下"的那个安静伙伴。请用陶土橘的温度、呼吸的节奏、留白的从容，把"今天，提了么？"变成一个让人微微一笑就想回答"提了"的问题。
